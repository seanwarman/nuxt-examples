# Containers

There are various container widths throughout the application. The default one has no `max-width` tailwind class applied and
instead uses the `.container` breakpoint widths to control its width at different breakpoints.

## Props

> styles: **object**[prop]

There's a prop on the container which is called `styles`. It accepts an object.

```
{
  width: 'small' | 'medium' | 'large' | 'fluid',
  padding: {
    top_padding: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
    bottom_padding: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
    x_padding: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
    y_padding: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
  },
  margin: {
    top_margin: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
    bottom_margin: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
    x_margin: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
    y_margin: 'small' | 'medium' | 'large' | 'xlarge' | 'none',
  }
}
```

## Default Container

Relies on the `.container` class for its max-widths. These values change based on which breakpoint the page is on.
